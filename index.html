Stage 0

Getting Started with HTML

5 Basics of the Web

  1. The Web is a collection of web pages written in a language called HTML. Web browsers read this documents and then display it to the user.
  2. HTML (Hyper Text Markup Language) document contains text and allows reference to other documents and links to other pages.
  3. HTTP is a type of communication method for the web browser to make requests via the internet to the servers.
  4. Web Applications: Will learn at the end of the course.

Helpful Reminders

  1. Programmers don't need to remember everything.
  2. Basic HTML vocabs:
    1. Most HTML tags require a closing tag with the exception of "void" tags.
    2. An element refers to everything within the opening and closing tags.
    3. An attribute or "a" is a property of an HTML element.
    4. An element can either be "block" or "inline", the difference between both is one can be styled via "CSS" and another does not.

Stage 1.2

Creating a Structured Document

  1. HTML and CSS change over time but the principles remains the same, therefore its important to understand how to break the page into smaller manageable chunks.
  2. Simple way to understand how the whole website components work is to think of it as a house:
    1. HTML is the structure where the wall is placed.
    2. CSS is the style of your house.
    3. Javascript is your interactive components; garage door, television remote and etc.
  3. There are rules for how websites behave.
  4. DOM (document object model) - standard convention to represent and interaction elements in HTML.
  5. DOM tree - HTML structure of the website.
  6. CSS - to define style of website.
  7. Every element on a website is boxed shape, regardless of how it "look", its core is a box shape.
  8. To dissect a website design is to break the site design into smaller boxes.
  9. Start breaking the website from big to small and to start building back from the reverse.
  10. Use div tags in code to represent a "block" in a website, this allows code to be read easier

Stage 1.3

Adding CSS Style and HTML Structure

  1. CSS (cascading style sheets) - can have multiple styles in a site.
  2. Cascading means that the rule applies to all, this is the idea known as inheritance and requires abstract thinking.
  3. Usage of specific code reference sites will help in the code development, it's important to know how to refer to these sites for information, eg: MDN CSS reference.
  4. The Box Model; margin (space between boxes), bodder, padding, content.
  5. Width of the element is actually sum of padding and content.
  6. *{box-sizing: border-box;} calculates the box size without including the margin.
  7. Box Sizing:
    1. HTML elements are boxes that each are made of 4 components (Box Model); margin, border, padding and content.
    2. Due to the many components to each box, there are 2 techniques to help deal with sizing issues:
      1. Set sizes in terms of percentages rather than pixels.
      2. Set "box-sizing" attribute to "border-box" for every element, this calculates the box size without including the margin.
      3. Different browsers work slightly differently hence why we need to add browser prefixed CSS tags.
  8. Box Positioning:
    1. Divs are block elements, hence they take up the entire width of a page.
    2. The rule "display:flex;" to the appropriate CSS overrides this behavior and let divs appear next to each other.
    3. Refer to this guide for more informationg regarding CSS flexbox.
  9. Avoid Repetitions during programming. In the example of web development, changing the style of an element is much faster and easier if similar elements shares the same class or id tags. As any elements that contains the same tags will apply the same style. More information regarding the differences or similarities of a class or id tags can be found here.
  10. Debugging: As with all things, mistakes do happen and even experience programmers at times make mistakes, therefore it's important to know how to debug the errors that we find in our code. In the case of web development a few things that we can do is as follows:
    1. Validate your HTML and CSS.
    2. Open and test website in different browsers to ensure it's rendered correctly, if there's a problem, use the respective web browser developer tools to debug the website. More information can be found here.
  11. Code, Test, Refine
    1. Look for natural boxes.
    2. Look for repeated styles and semantic elements.
    3. Write your HTML.
    4. Apply styles (from BIGGEST to smallest).
    5. Fix things.
